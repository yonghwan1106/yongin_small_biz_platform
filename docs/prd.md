## 1. 개요

### 1.1. 문제 정의

#### 1.1.1. 배경
한국의 소상공인은 전체 사업체의 약 88%를 차지하며, 약 630만 개의 사업장에서 1,500만 명 이상이 종사하고 있습니다. 특히 용인시는 약 10만 개 이상의 소상공인 사업장이 운영되고 있으며, 이들은 지역 경제의 핵심 축을 담당하고 있습니다.

#### 1.1.2. 핵심 문제점
1. **정보 접근성의 격차**
   - 대기업/프랜차이즈: 전문 상권 분석 솔루션, 데이터 애널리스트 팀 보유
   - 소상공인: 월 수십만 원의 비용이 드는 상권 분석 서비스 이용 어려움
   - 결과: 직관과 경험에만 의존한 의사결정 → 매출 변동성 증가

2. **데이터 활용 역량 부족**
   - 경기도는 풍부한 공공 데이터를 제공하지만, 이를 해석하고 활용할 수 있는 소상공인은 극소수
   - 원시 데이터는 복잡하고 전문 지식이 필요해 실질적 활용이 어려움

3. **시의성 있는 대응 어려움**
   - 상권 변화를 감지하는 시점이 늦어 매출 하락 후에야 문제를 인지
   - 유동인구 패턴 변화, 소비 트렌드 전환 등에 대한 조기 경보 시스템 부재

#### 1.1.3. 비즈니스 임팩트
- **재고 손실**: 수요 예측 실패로 인한 평균 15-20% 재고 폐기율
- **인력 비효율**: 피크 타임 예측 실패로 인한 과소/과다 인력 배치
- **마케팅 ROI 저하**: 타겟 고객층 및 최적 타이밍 파악 실패로 전단지, 쿠폰 등의 효과 30% 이하
- **기회 손실**: 상권 변화에 따른 메뉴/상품 조정 지연으로 평균 월 매출의 10-15% 손실 추정

### 1.2. 제품 비전

**"모든 용인 소상공인이 대기업 수준의 데이터 인사이트로 스마트하게 경쟁할 수 있는 세상"**

'용인 소상공인 활력 지수'는 경기도의 공공 데이터를 활용하여, 용인시 소상공인에게 자신의 상권에 대한 깊이 있는 통찰력을 제공하는 **AI 기반 상권 분석 SaaS 플랫폼**입니다. 

#### 핵심 가치 제안 (Value Proposition)
1. **접근성**: 무료 또는 저비용으로 전문가 수준의 상권 분석 제공
2. **단순성**: 복잡한 데이터를 일반인도 이해 가능한 자연어 인사이트로 변환
3. **실행 가능성**: 단순 정보 제공이 아닌, 구체적이고 즉시 실행 가능한 비즈니스 제안
4. **지속성**: 일회성이 아닌, 지속적인 모니터링과 트렌드 추적

### 1.3. 목표 시장

#### 1.3.1. 주요 타겟 (Primary Target)
- **지역**: 경기도 용인시 (처인구, 기흥구, 수지구)
- **업종**: 외식업(카페, 레스토랑, 주점), 소매업(의류, 잡화, 편의점), 서비스업(미용실, 네일샵, 학원)
- **규모**: 연 매출 1억~10억 원, 직원 수 1~10명
- **운영 기간**: 창업 1년 이상 (기본적인 비즈니스 운영 경험 보유)

#### 1.3.2. 시장 규모 (TAM, SAM, SOM)
- **TAM (Total Addressable Market)**: 한국 전체 소상공인 약 630만 개
- **SAM (Serviceable Addressable Market)**: 경기도 소상공인 약 100만 개
- **SOM (Serviceable Obtainable Market)**: 용인시 소상공인 약 10만 개
  - 초기 목표: 외식/소매/서비스업 중심 약 3만 개 사업장

#### 1.3.3. 2차 타겟 (Secondary Target)
- 창업 준비자 (상권 사전 조사용)
- 소상공인 지원 기관 (용인시청, 소상공인진흥공단 등)
- 상권 컨설턴트 및 부동산 중개업체

### 1.4. 제품 차별화 요소

| 요소 | 기존 솔루션 | 용인 소상공인 활력 지수 |
|------|-------------|------------------------|
| **가격** | 월 10-50만 원 | 무료 (프리미엄: 월 2-3만 원) |
| **사용 난이도** | 전문가 수준 필요 | 누구나 이해 가능 |
| **인사이트 제공** | 원시 데이터 중심 | AI 자연어 해석 및 제안 |
| **개인화** | 일반적인 상권 정보 | 내 가게 맞춤형 분석 |
| **업데이트 주기** | 월간 또는 수동 | 주간 자동 리포트 |
| **지역 특화** | 전국 단위 일반화 | 용인시 특화 데이터 |

## 2. 시장 분석 및 경쟁사 조사

### 2.1. 시장 트렌드

#### 2.1.1. 데이터 기반 의사결정 확산
- 코로나19 이후 오프라인 매장의 생존 전략으로 데이터 분석 중요성 급증
- '디지털 전환'에 대한 소상공인의 관심도 2020년 대비 300% 증가 (키워드 검색량 기준)

#### 2.1.2. 공공데이터 개방 가속화
- 경기도, 서울시 등 광역 지자체의 실시간 유동인구, 소비 데이터 공개
- 국가 차원의 '데이터 댐' 정책으로 공공데이터 품질 및 접근성 향상

#### 2.1.3. AI 기술 대중화
- ChatGPT 이후 생성형 AI에 대한 인식 개선
- 자연어 기반 인사이트 제공이 일반 사용자에게도 친숙해짐

### 2.2. 경쟁사 분석

#### 2.2.1. 직접 경쟁사

**A. 상권정보시스템 (소상공인시장진흥공단)**
- **강점**: 
  - 정부 지원으로 무료 제공
  - 전국 단위 방대한 데이터
  - 공신력
- **약점**:
  - UI/UX가 복잡하고 직관적이지 않음
  - 데이터 해석을 사용자가 직접 해야 함
  - 실시간성 부족 (분기별 업데이트)
  - 개인화된 인사이트 부재

**B. 상권 분석 앱 (민간 서비스)**
- **강점**:
  - 모바일 최적화
  - 깔끔한 UI
- **약점**:
  - 월 10-30만 원의 유료 구독
  - 일반화된 분석 (내 가게 특화 X)
  - AI 인사이트 부재

#### 2.2.2. 간접 경쟁사
- 네이버 플레이스/카카오맵의 통계 기능
- 지역 상공회의소/진흥원의 오프라인 컨설팅

#### 2.2.3. 경쟁 우위 전략
1. **하이퍼 로컬 전략**: 용인시 데이터에 집중하여 깊이 있는 분석
2. **AI 자연어 해석**: 복잡한 데이터를 평문으로 쉽게 설명
3. **실행 가능한 제안**: 단순 정보가 아닌 '무엇을 해야 하는지' 명확한 액션 아이템 제공
4. **지속적 관계**: 일회성 조회가 아닌 주간 리포트를 통한 장기적 관계

## 3. 목표 및 성공 지표

### 3.1. 사용자 목표

#### 3.1.1. 단기 목표 (사용 후 1개월 이내)
- [ ] 내 가게 주변 상권의 유동인구 규모와 패턴을 이해한다
- [ ] 피크 타임과 오프 타임을 데이터로 확인한다
- [ ] 주 타겟 고객층(연령대, 성별)을 파악한다

#### 3.1.2. 중기 목표 (3개월 이내)
- [ ] 데이터 기반으로 프로모션 시기와 타겟을 설정한다
- [ ] 재고 및 인력 배치를 유동인구 예측에 맞춰 조정한다
- [ ] 주간 트렌드 변화를 추적하고 대응 전략을 수립한다

#### 3.1.3. 장기 목표 (6개월 이상)
- [ ] 데이터 기반 의사결정이 일상화되어 매출 안정성이 향상된다
- [ ] 계절성, 이벤트 등 외부 요인의 영향을 사전에 예측하고 대비한다
- [ ] 사업 확장 또는 2호점 개점 시 상권 분석에 활용한다

### 3.2. 비즈니스 목표

#### 3.2.1. Phase 1: MVP 검증 (출시 후 3개월)
- **사용자 확보**: 
  - 총 가입자 500명
  - 활성 사용자(MAU) 300명 (60% 활성화율)
  
- **참여도**:
  - 주간 재방문율 40%
  - 주간 AI 리포트 조회율 60%
  - 평균 세션 시간 5분 이상

- **만족도**:
  - NPS(Net Promoter Score) 40 이상
  - 사용자 설문에서 80% 이상 긍정 평가
  - 주간 AI 리포트 유용성 평가 4.0/5.0 이상

#### 3.2.2. Phase 2: 성장 (출시 후 6개월)
- 총 가입자 2,000명
- MAU 1,200명 (60% 유지)
- 프리미엄 전환율 5% (60명)
- 월간 매출 180만 원 (프리미엄 구독)

#### 3.2.3. Phase 3: 확장 (출시 후 1년)
- 총 가입자 5,000명
- MAU 3,000명
- 프리미엄 전환율 10% (500명)
- 월간 매출 1,500만 원
- 타 지역(성남시, 수원시) 확장 준비

### 3.3. 핵심 지표 (Key Metrics)

#### 3.3.1. 획득 (Acquisition)
- **유입 채널별 가입자 수**
  - 측정 방법: Google Analytics + UTM 파라미터
  - 목표: 유기적 검색 40%, 소셜미디어 30%, 제휴 20%, 직접 방문 10%

- **가입 전환율**
  - 측정 방법: (가입 완료 수 / 랜딩 페이지 방문 수) × 100
  - 목표: 15% 이상

#### 3.3.2. 활성화 (Activation)
- **온보딩 완료율**
  - 정의: 가입 후 가게 위치 등록 + 첫 대시보드 조회
  - 측정 방법: Funnel 분석
  - 목표: 80% 이상

- **첫 주간 리포트 조회율**
  - 정의: 가입 후 첫 번째 주간 리포트 수신 시 48시간 내 조회
  - 목표: 70% 이상

#### 3.3.3. 유지 (Retention)
- **Week 1 Retention**: 70%
- **Week 4 Retention**: 40%
- **Week 12 Retention**: 25%

- **주간 활성 사용자 (WAU)**
  - 정의: 7일 내 1회 이상 로그인
  - 목표: 전체 가입자의 50%

#### 3.3.4. 수익 (Revenue) - Phase 2 이후
- **프리미엄 전환율**: 5-10%
- **평균 구독 유지 기간**: 6개월 이상
- **ARPU (Average Revenue Per User)**: 월 3,000원

#### 3.3.5. 추천 (Referral)
- **NPS (Net Promoter Score)**: 40 이상
- **바이럴 계수 (K-factor)**: 0.3 이상
- **입소문을 통한 신규 가입 비율**: 20%

### 3.4. 데이터 수집 및 분석 도구
- **웹 분석**: Google Analytics 4
- **사용자 행동 추적**: Mixpanel (또는 Amplitude)
- **A/B 테스트**: Google Optimize
- **세션 리플레이**: Hotjar
- **설문조사**: Google Forms + 인앱 설문

**User Story**

**기능 요구사항**

- 이메일 중복 체크
- 비밀번호 강도 검증 (최소 8자, 영문+숫자 조합)
- 이메일 인증 (선택적 - Phase 2)
- 소셜 로그인 (네이버, 카카오) - Phase 2

**UI/UX 요구사항**
```
[화면 구성]
1. 랜딩 페이지에서 "무료로 시작하기" 버튼
2. 회원가입 모달 또는 별도 페이지
   - 이메일 입력 필드
   - 비밀번호 입력 필드
   - 비밀번호 확인 필드
   - 이용약관 동의 체크박스 (필수)
   - 개인정보처리방침 동의 체크박스 (필수)
   - 마케팅 수신 동의 체크박스 (선택)
   - "가입하기" 버튼
3. 가입 완료 후 → 온보딩 플로우로 자동 이동
```

**Acceptance Criteria**
- [ ] 유효한 이메일 형식만 입력 가능
- [ ] 이미 가입된 이메일은 재가입 불가 (에러 메시지 표시)
- [ ] 비밀번호 강도 조건 미충족 시 실시간 피드백
- [ ] 필수 약관 미동의 시 가입 버튼 비활성화
- [ ] 가입 성공 시 자동 로그인 + 온보딩 페이지 이동
- [ ] 가입 실패 시 명확한 에러 메시지 표시

**기술 명세**
- **API Endpoint**: `POST /api/auth/signup`
- **Request Body**:
```json
{
  "email": "user@example.com",
  "password": "securePassword123",
  "agreements": {
    "terms": true,
    "privacy": true,
    "marketing": false
  }
}
```
- **Response** (성공):
```json
{
  "success": true,
  "userId": "usr_abc123",
  "token": "jwt_token_here",
  "message": "회원가입이 완료되었습니다."
}
```
- **Response** (실패):
```json
{
  "success": false,
  "error": "EMAIL_ALREADY_EXISTS",
  "message": "이미 가입된 이메일입니다."
}
```

| 컬럼명 | 타입 | 예시 | 비고 |

| createdAt | Timestamp | 2025-10-24T10:30:00Z | ISO 8601 |
| lastLoginAt | Timestamp | 2025-10-24T10:30:00Z | |
| isEmailVerified | Boolean | false | Phase 2 |

##### B. 상권 활력 지수 히트맵 [P0]

**목적**
내 가게 주변 상권의 활력도를 시각적으로 직관적으로 이해

**데이터 처리 로직**
```python
def calculate_vitality_index(location, radius=500):
    """
    활력 지수 = (유동인구 점수 × 0.4) + (소비액 점수 × 0.4) + (성장률 점수 × 0.2)
    
    Parameters:
    - location: (lat, lng) 튜플
    - radius: 분석 반경 (미터)
    
    Returns:
    - vitality_index: 0-100 사이의 점수
    - heatmap_data: 지도에 표시할 그리드 데이터
    """
    
    # 1. 반경 내 유동인구 데이터 가져오기
    foot_traffic = get_foot_traffic(location, radius)
    foot_traffic_score = normalize_score(foot_traffic, max_value=10000)
    
    # 2. 소비액 데이터
    consumption = get_consumption_data(location, radius)
    consumption_score = normalize_score(consumption, max_value=50000000)
    
    # 3. 성장률 (전주 대비)
    growth_rate = calculate_growth_rate(location, weeks=4)
    growth_score = normalize_score(growth_rate, min_value=-20, max_value=20)
    
    # 최종 점수 계산
    vitality_index = (
        foot_traffic_score * 0.4 +
        consumption_score * 0.4 +
        growth_score * 0.2
    ) * 100
    
    # 히트맵용 그리드 데이터 생성 (100m×100m 그리드)
    heatmap_data = generate_heatmap_grid(location, radius, vitality_index)
    
    return {
        'index': round(vitality_index, 1),
        'heatmap': heatmap_data,
        'breakdown': {
            'foot_traffic': foot_traffic_score,
            'consumption': consumption_score,
            'growth': growth_score
        }
    }
```

**API 엔드포인트**
```javascript
// pages/api/cron/generate-weekly-reports.js
import { getActiveUsers } from '@/lib/database';
import { fetchWeeklyData } from '@/lib/data-aggregator';
import { generateReport } from '@/lib/anthropic';
import { saveReport } from '@/lib/database';

export default async function handler(req, res) {
  // Cron secret으로 인증 (보안)
  if (req.headers.authorization !== `Bearer ${process.env.CRON_SECRET}`) {
    return res.status(401).json({ error: 'Unauthorized' });
  }

  try {
    const users = await getActiveUsers();
    const reports = [];

    for (const user of users) {
      // 1. 데이터 수집
      const weeklyData = await fetchWeeklyData(user.location, user.category);
      
      // 2. AI 보고서 생성
      const report = await generateReport({
        user: user,
        data: weeklyData,
        model: 'claude-sonnet-4-5-20250929'
      });
      
      // 3. 저장
      await saveReport({
        userId: user.id,
        reportDate: new Date().toISOString(),
        content: report,
        dataSnapshot: weeklyData
      });

      reports.push({ userId: user.id, status: 'success' });
    }

    res.status(200).json({ 
      success: true, 
      generatedReports: reports.length,
      details: reports
    });

  } catch (error) {
    console.error('Report generation failed:', error);
    res.status(500).json({ error: error.message });
  }
}
```

**Anthropic API 호출 함수**
```javascript
// lib/anthropic.js
import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
});

export async function generateReport({ user, data, model }) {
  const systemPrompt = `당신은 소상공인을 위한 상권 분석 전문가입니다...`;
  
  const userPrompt = `
다음은 ${user.name}님 가게의 지난주 상권 데이터입니다.

[가게 정보]
- 이름: ${user.storeName}
- 업종: ${user.category}
- 위치: ${user.address}

[지난주 데이터]
- 일평균 유동인구: ${data.avgFootTraffic}명 (전주 대비 ${data.growthRate}%)
- 일평균 소비액: ${data.avgConsumption}원
...
`;

  const message = await client.messages.create({
    model: model,
    max_tokens: 2048,
    system: systemPrompt,
    messages: [
      { role: 'user', content: userPrompt }
    ],
  });

  return message.content[0].text;
}
```

## 12. 출시 전략

### 12.1. 타겟 시장 진입 전략

#### 12.1.1. Phase 1: 얼리어답터 확보 (0-3개월)
- **타겟**: IT 친화적 젊은 창업자, 데이터에 관심 있는 소상공인

  - 용인시 청년 창업 지원센터 협력

- **메시지**: "대기업만 쓰던 상권 분석, 이제 무료로!"

#### 12.1.2. Phase 2: 주류 시장 확대 (3-6개월)
- **타겟**: 일반 소상공인 (40-50대)
- **채널**:
  - 용인시청 소상공인과 협력 공지

#### 12.1.3. Phase 3: 지역 표준화 (6-12개월)
- **목표**: 용인 소상공인 10명 중 1명이 사용
- **전략**:
  - 성공 사례 발굴 및 홍보
  - 용인시 공식 추천 도구로 인증
  - 소상공인 지원 정책과 연계

## 14. 부록

### 14.1. 용어 정의 (Glossary)

| 용어 | 정의 |
|------|------|
| **상권 활력 지수** | 유동인구, 소비액, 성장률을 종합한 0-100점의 상권 활성도 지표 |
| **유동인구** | 특정 지역을 방문하거나 통과하는 사람의 수 (경기도 공공데이터 기준) |
| **피크 타임** | 유동인구 또는 소비가 가장 많은 시간대 |
| **히트맵** | 데이터 밀도를 색상으로 표현한 지도 시각화 (빨강: 높음, 파랑: 낮음) |
| **MVP** | Minimum Viable Product, 최소 기능 제품 |
| **PMF** | Product-Market Fit, 제품-시장 적합성 |
| **NPS** | Net Promoter Score, 순 추천 고객 지수 (-100 ~ +100) |
| **WAU/MAU** | Weekly/Monthly Active Users, 주간/월간 활성 사용자 |
| **Churn Rate** | 이탈률, 서비스를 그만두는 사용자 비율 |
